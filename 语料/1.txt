# “洞居”用户流程痛点分析、AI解决方案暨AI所需数据

# C端是智能体数据资源流量入口，B端是智能体用户付费端，B端用户更接近家居行业市场，与“洞居”标的接触时间更长，接触深度更深，付费意愿更强，C端用户基数大，数据体量大。

## C端

### 一、注册环节

| 痛点 | AI解决方案 | 所需数据 |
| --- | --- | --- |
| 1.1 房产认证流程繁琐 | OCR识别房产证/购房合同，自动提取房产信息；LBS智能推荐小区 | 房产证/购房合同样本图像数据集（用于OCR模型训练）； 全国小区地址库（小区名称、地址、楼栋信息）； 用户GPS定位数据 |
| 1.2 用户隐私顾虑 | AI隐私顾问：智能问答解答隐私疑虑，可视化展示数据脱敏流程 | 隐私相关FAQ问答对数据集；数据脱敏规则库；用户隐私咨询历史记录 |
| 1.3 平台价值认知不清 | 个性化收益预测：基于房产面积、装修预算，AI计算预期补贴收益 | 历史用户补贴收益数据（按面积、预算、品类分布）；各品类平均消费金额统计；补贴规则参数配置表 |
| 1.4 房产真实性验证困难 | 多模态房产核验：结合OCR、人脸识别、地理位置多维度验证 | 房产证OCR标注数据；人脸-身份证匹配数据；房产登记信息API（如可接入）；小区地理围栏数据 |

### 二、记录装修"想法"环节

| 痛点 | AI解决方案 | 所需数据 |
| --- | --- | --- |
| 2.1 不知道该发布哪些需求 | 智能需求引导：AI根据户型、面积、风格偏好，生成装修品类清单 | 户型-品类关联矩阵（不同户型常用品类）；装修风格-材料映射表；面积-预算-品类推荐模型训练数据 |
| 2.2 需求描述不专业 | 需求描述优化：NLP模型自动补全、润色用户输入 | 装修领域专业术语词典；用户原始描述→标准化描述配对数据；商家偏好的需求描述样本 |
| 2.3 不清楚什么"想法"更有价值 | 实时价值评估：AI预测每条需求的潜在收益 | 历史需求-商家购买转化率数据；各品类需求的市场供需数据；商家出价历史记录 |
| 2.4 手动发布效率低 | 语音/图片快捷发布：语音转文字、图片识别批量生成需求 | 装修领域语音识别训练数据（方言、专业术语）；家具/建材图像分类数据集；样板间/效果图标注数据 |
| 2.5 历史需求难以管理 | 智能需求管理：AI定期提醒更新需求状态 | 用户需求生命周期数据（从发布到完成的时间分布）；装修工序时间线模板；用户行为日志（登录、操作记录） |

### 三、逛店比价环节

| 痛点 | AI解决方案 | 所需数据 |
| --- | --- | --- |
| 3.1 不知道该去哪些店 | 智能逛店规划：AI根据需求、位置、评分生成最优逛店路线 | 商家POI数据（位置、品类、营业时间）；商家评分/评价数据；用户-商家匹配历史数据；地图路径规划API |
| 3.2 线下逛店耗时耗力 | AR虚拟逛店：AI+AR技术支持线上3D展厅浏览 | 商家门店3D扫描数据；产品3D模型库；商品信息结构化数据 |
| 3.3 到店忘记出示优惠券 | LBS智能提醒：进入商圈/门店范围时自动推送提醒 | 商家地理围栏数据（经纬度、覆盖半径）；用户实时位置数据；用户优惠券持有状态 |
| 3.4 商家核销流程不顺畅 | 智能核销助手：AI语音助手引导核销，异常自动客服介入 | 核销流程SOP知识库；常见核销问题FAQ；核销异常case数据集 |
| 3.5 逛店信息记录混乱 | AI比价助手：自动记录各商家报价，生成对比表格 | 用户逛店记录（商家、时间、咨询内容）；商家报价结构化模板；产品规格标准化字典 |
| 3.6 难以判断报价是否合理 | 价格合理性评估：基于大数据分析判断报价区间 | 各品类历史成交价格数据；区域价格指数；品牌-档次-价格映射表；原材料价格波动数据 |

### 四、完成交易环节

| 痛点 | AI解决方案 | 所需数据 |
| --- | --- | --- |
| 4.1 交易金额填写可能出错 | OCR票据识别：拍照上传收据/合同，AI自动识别交易金额 | 收据/发票/合同OCR标注数据集；金额字段定位模型训练数据；各商家票据模板库 |
| 4.2 商家扫码配合度不一 | 商家端AI培训：智能客服为商家提供核销流程指导 | 商家操作培训视频/文档；商家常见问题FAQ；商家操作行为日志 |
| 4.3 补贴到账时间不透明 | 智能进度追踪：AI实时推送审核进度，预测到账时间 | 补贴审核流程节点数据；历史审核时长统计；影响审核时长的因素数据 |
| 4.4 交易纠纷缺乏凭证 | AI证据链管理：自动归档交易凭证，纠纷时生成证据报告 | 交易全流程日志（时间戳、操作记录）；聊天记录、图片、合同等凭证；纠纷案例库及处理结果 |
| 4.5 多品类交易管理复杂 | 补贴管家：AI追踪所有品类交易状态，提醒未申领补贴 | 用户装修品类清单；各品类交易状态数据；补贴申领规则及截止时间 |

### 五、整体平台体验

| 痛点 | AI解决方案 | 所需数据 |
| --- | --- | --- |
| 5.1 装修周期长易流失 | AI装修管家：根据装修进度智能推送任务提醒 | 标准装修工序时间线模板；用户装修进度数据；用户活跃度/留存数据；推送效果反馈数据 |
| 5.2 补贴规则复杂 | 收益计算器：AI实时汇总补贴，可视化展示收益明细 | 完整补贴规则配置表；用户历史补贴明细；各类补贴计算公式 |
| 5.3 缺乏装修全流程指导 | AI装修顾问：基于大模型的装修问答助手 | 装修知识库（选材、工序、避坑指南）；装修问答对数据集；行业标准/规范文档；用户真实装修案例 |
| 5.4 平台信任度建立困难 | 透明化AI解读：可视化解释商业模式和数据处理流程 | 商业模式说明文档；数据流转流程图；用户信任度调研数据；竞品对比分析 |
| 5.5 用户间缺乏交流 | AI社区运营：智能匹配同小区业主，AI总结优质经验帖 | 用户画像数据（小区、户型、装修阶段）；社区帖子内容数据；帖子互动数据（点赞、评论、收藏）；用户关系图谱 |

### C端数据采集与建设优先级建议

### 第一优先级（核心业务数据）

| 数据类型 | 来源 | 用途 |
| --- | --- | --- |
| 用户交易数据 | 平台自有 | 收益预测、价格评估、补贴计算 |
| 商家信息数据 | 平台自有+爬取 | 商家推荐、逛店规划 |
| 用户行为日志 | 平台埋点 | 智能提醒、流失预警 |
| 补贴规则配置 | 业务系统 | 收益计算器 |

### 第二优先级（AI模型训练数据）

| 数据类型 | 来源 | 用途 |
| --- | --- | --- |
| 房产证/票据图像 | 用户上传+标注 | OCR模型训练 |
| 装修问答对 | 人工整理+用户反馈 | 装修顾问大模型微调 |
| 需求描述配对 | 历史数据清洗 | NLP需求优化模型 |
| 家具建材图像 | 公开数据集+采集 | 图像识别模型 |

### 第三优先级（增强体验数据）

| 数据类型 | 来源 | 用途 |
| --- | --- | --- |
| 3D门店/产品模型 | 商家提供+扫描 | AR虚拟逛店 |
| 装修案例库 | UGC+PGC  | 社区运营、案例推荐 |
| 市场价格指数 | 行业报告+爬取 | 价格合理性评估 |

## B端

### B端商家核心流程概述

### 入驻认证 → 购买数据产品 → 获客触达 → 到店接待 → 交易转化 → 核销结算 → 经营分析

### 一、入驻认证环节

| 痛点 | AI解决方案 | 所需数据 |
| --- | --- | --- |
| 1.1 入驻资质审核周期长，材料反复修改 | 智能资质预审：AI预检营业执照、资质证书，实时反馈问题并指导修正 | 营业执照OCR标注数据集；各品类资质要求规则库；历史审核驳回原因数据 |
| 1.2 店铺信息填写繁琐（地址、品类、品牌等） | 智能信息补全：基于营业执照自动提取企业信息，结合地图API补全地址 | 企业工商信息API；地图POI数据；品类-品牌映射表 |
| 1.3 不清楚平台规则和收费模式 | AI入驻顾问：智能问答解答入驻疑问，个性化推荐适合的合作方案 | 平台规则知识库；收费方案配置表；商家入驻FAQ数据集；同品类商家合作案例 |
| 1.4 竞品商家已入驻，担心市场饱和 | 市场机会分析：AI分析该区域该品类的供需比，展示潜在客户量 | 区域用户需求分布数据；已入驻商家密度数据；品类竞争度指数 |

### 二、购买数据产品环节

| 痛点 | AI解决方案 | 所需数据 |
| --- | --- | --- |
| 2.1 不知道该买哪些数据产品，担心ROI低 | 智能选品推荐：AI根据商家品类、位置、预算，推荐高转化潜力的数据包 | 商家画像数据（品类、客单价、服务范围）；数据产品历史转化率；用户需求-商家匹配模型 |
| 2.2 数据产品定价不透明，难以评估价值 | 数据价值评估器：AI展示该数据包的预期触达量、历史转化率、同行购买情况 | 数据产品定价规则；历史购买-转化数据；同品类商家购买行为数据 |
| 2.3 购买后发现用户需求与自身业务不匹配 | 需求匹配度预测：购买前AI预测该批用户与商家的匹配度评分 | 用户需求详情（预算、风格、品牌偏好）；商家服务能力标签；历史匹配成功案例 |
| 2.4 批量购买时难以筛选优质线索 | AI线索评分：对每条用户需求进行意向度、消费能力、紧迫度评分 | 用户行为数据（浏览、咨询、逛店频次）；用户装修进度数据；历史线索转化标注数据 |
| 2.5 数据时效性担忧，怕买到"过期"需求 | 需求新鲜度标识：AI标注需求发布时间、用户活跃度、是否已被其他商家触达 | 需求发布时间戳；用户最近活跃时间；需求被购买/触达次数 |

### 三、获客触达环节

| 痛点 | AI解决方案 | 所需数据 |
| --- | --- | --- |
| 3.1 不知道如何有效触达用户，话术不专业 | 智能话术生成：AI根据用户需求特点，生成个性化触达话术 | 高转化话术模板库；用户需求关键词；A/B测试话术效果数据 |
| 3.2 触达时机不当，用户响应率低 | 最佳触达时机预测：AI分析用户活跃时段，推荐最佳联系时间 | 用户APP活跃时段数据；历史触达-响应时间关联数据；用户时区/作息偏好 |
| 3.3 批量触达效率低，人工成本高 | AI外呼/消息助手：智能批量触达，自动记录用户反馈，筛选高意向用户 | 用户联系方式（授权后）；外呼话术脚本；用户反馈分类标签 |
| 3.4 用户已被多家商家骚扰，产生抵触 | 触达频控管理：AI控制单用户被触达频次，避免过度营销 | 用户被触达历史记录；用户投诉/拉黑数据；行业触达频次基准 |
| 3.5 难以追踪触达效果，无法优化策略 | 触达效果分析：AI自动统计各渠道、各话术的转化漏斗 | 触达渠道数据（电话、短信、站内信）；各环节转化率数据；用户反馈文本数据 |

### 四、到店接待环节

| 痛点 | AI解决方案 | 所需数据 |
| --- | --- | --- |
| 4.1 用户到店前不了解其需求，接待准备不足 | 客户画像预览：用户预约到店时，AI推送该用户的需求摘要和偏好分析 | 用户需求详情；用户浏览/收藏记录；用户历史咨询记录 |
| 4.2 到店用户身份核验麻烦 | 智能身份核验：用户出示平台码，AI自动匹配用户信息并提示接待要点 | 用户-优惠券绑定数据；用户预约记录；商家员工权限数据 |
| 4.3 接待过程中难以记录用户反馈 | AI接待助手：语音实时转文字，自动提取用户关注点和异议 | 接待对话录音（用户授权）；销售话术知识库；用户异议分类标签 |
| 4.4 店员销售能力参差不齐 | AI销售教练：实时提示销售话术，针对用户异议给出应对建议 | 优秀销售话术库；常见异议应对方案；店员销售数据（转化率、客单价） |
| 4.5 用户逛完就走，缺乏跟进机制 | 智能跟进提醒：AI分析用户离店后的意向度，推荐跟进策略和时机
 | 用户到店时长数据；用户咨询深度数据；历史跟进-成交关联数据 |

### 五、交易转化环节

| 痛点 | AI解决方案 | 所需数据 |
| --- | --- | --- |
| 5.1 报价后用户犹豫，不知如何促单 | AI促单助手：分析用户犹豫原因，推荐针对性促单策略（优惠、赠品、分期等） | 用户价格敏感度数据；促单策略效果数据；竞品价格情报 |
| 5.2 用户比价后流失，不知输在哪里 | 竞争分析报告：AI分析流失用户最终选择的商家，找出竞争差距 | 用户最终成交商家数据；竞品商家画像；用户流失原因调研数据 |
| 5.3 合同签订流程繁琐，影响成交效率 | 智能合同生成：AI根据交易内容自动生成合同，支持电子签约 | 合同模板库；交易明细结构化数据；电子签约API |
| 5.4 定金/尾款收取管理混乱 | AI收款管家：自动生成收款计划，提醒收款节点，追踪回款状态 | 交易付款条款；收款记录数据；用户付款行为数据 |
| 5.5 售后承诺口头化，易产生纠纷 | 承诺记录器：AI自动记录销售过程中的承诺内容，生成书面确认  | 销售对话记录；承诺内容提取模型；售后服务标准库 |

### 六、核销结算环节

| 痛点 | AI解决方案 | 所需数据 |
| --- | --- | --- |
| 6.1 核销流程不熟悉，操作出错 | 智能核销引导：AI语音/图文引导完成核销，异常自动提示解决方案 | 核销操作SOP；常见核销错误及解决方案；商家操作日志 |
| 6.2 与平台结算周期长，现金流压力大 | 结算预测器：AI预测结算到账时间，提供结算进度实时查询  | 结算规则配置；历史结算周期数据；财务审核流程节点 |
| 6.3 结算金额有争议，对账困难 | AI对账助手：自动生成交易明细对账单，差异项智能标注 | 交易流水数据；平台抽成规则；优惠券核销记录 |
| 6.4 发票开具管理混乱 | 智能发票管理：AI自动汇总开票需求，生成开票清单，对接电子发票系统 | 交易开票信息；商家税务信息；电子发票API |

### 七、经营分析环节

| 痛点 | AI解决方案 | 所需数据 |
| --- | --- | --- |
| 7.1 不清楚在平台的投入产出比 | ROI分析看板：AI自动计算数据购买成本、获客成本、成交收益，生成ROI报告 | 数据购买支出记录；获客-成交转化数据；交易金额数据 |
| 7.2 不知道如何优化获客策略 | AI经营顾问：基于数据分析，给出获客策略优化建议（调整品类、区域、预算等） | 商家历史经营数据；同行业标杆数据；市场趋势数据 |
| 7.3 缺乏用户评价管理，口碑难以积累 | 评价管理助手：AI监控用户评价，负面评价预警，生成回复建议 | 用户评价数据；评价情感分析模型；优质回复模板库 |
| 7.4 难以了解市场趋势和竞争态势 | 市场情报分析：AI定期推送区域装修市场趋势、竞品动态、用户需求变化 | 区域装修需求趋势数据；竞品商家动态；行业报告数据 |
| 7.5 员工绩效难以量化考核 | 员工绩效分析：AI统计各员工的线索跟进量、转化率、客单价等指标  | 员工操作日志；员工-客户关联数据；成交归因数据 |

### B端数据采集与建设优先级建议

### 第一优先级（核心业务数据）

| 数据类型 | 来源 | 用途 |
| --- | --- | --- |
| 商家交易数据 | 平台自有 | ROI分析、结算对账 |
| 数据产品购买记录 | 平台自有 | 选品推荐、价值评估 |
